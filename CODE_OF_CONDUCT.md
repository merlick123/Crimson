# Code of Conduct

## Our Standard

Participation in this project should remain respectful, direct, and constructive.

Examples of expected behavior:

- giving and receiving technical feedback professionally
- focusing criticism on code, design, and evidence
- assuming good faith while still being rigorous
- helping keep discussions actionable

Examples of unacceptable behavior:

- harassment or personal attacks
- discriminatory language or conduct
- deliberate intimidation
- bad-faith disruption of project work

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, and pull requests that violate this code of conduct.

## Reporting

Report issues through the security/contact channel described in `SECURITY.md`.
